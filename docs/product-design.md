# プロダクト設計

## 目的

Open-Meteo の Forecast API を利用し、都市・指標・期間を選択式UIで指定することで、時系列の天気情報をチャートとして可視化する1ページ完結のSPAを構築する。

---

## 機能要件

### 都市選択

- 東京・大阪・札幌・福岡・那覇の5都市をセレクトボックスで選択できる
- 選択都市に紐づいた緯度・経度を内部参照してAPIをコールする

### 指標選択

- hourly: 気温（temperature_2m）・体感温度（apparent_temperature）・降水量（precipitation）・風速（windspeed_10m）
- daily: 最高気温（temperature_2m_max）・最低気温（temperature_2m_min）
- チェックボックス等で複数同時選択し、多系列重ね描きに対応する

### 期間切り替え

- 48時間（hourly）と7日間（daily or hourly集計）をトグルで切り替えられる

### 単位切り替え

- 気温は °C / °F をトグルで切り替えられる
- 風速単位（m/s・km/h 等）を切り替えられる

### チャート表示

- Recharts の折れ線グラフで描画する
- ツールチップ・凡例・Y軸単位・X軸（時刻/日付）フォーマットを表示する
- 複数指標選択時は多系列で重ね描きする

### 選択状態の永続化

- 都市・指標・期間・単位の直近選択を localStorage に保存し、再訪時に復元する

---

## 非機能要件

### データ取得・キャッシュ

- TanStack Query（useQuery）でフェッチ・キャッシュ・ローディング/エラー状態を一元管理する
- 選択変更時のみフェッチし、同一パラメータへの重複リクエストはキャッシュで吸収する
- デバウンスを実装し、Open-Meteo のレート制限（日1万/時5千/分600リクエスト）に配慮する

### UX

- ローディング中はスピナーまたはスケルトンプレースホルダを表示する
- エラー時（ネットワーク障害・レート超過等）はリトライボタンを伴うメッセージを表示する
- 指標が未選択の場合は空状態のメッセージを表示する

### アクセシビリティ

- セレクト・トグル・チェックボックスに適切な `<label>` を付与する

### ブラウザ対応

- Google Chrome 最新版で正しく動作する
